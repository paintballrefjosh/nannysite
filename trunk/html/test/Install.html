Menu triggered by an anchor

If you want a menu to be triggered by the mouse over of an anchor you have to create an anchor in your document with the syntax:

<a href='mylink.htm' onmouseover="PopMenu('PopMenuX',event)" onmouseout="OutMenu('PopMenuX')">MyAnchorText</a>

X in PopMenuX is the number of the PopMenu you use with that anchor.
 
Normally you will want an anchor triggered menu to be invisible untill the mouse is over the anchor.
You achieve that by setting PermanentVisible to 0;
 
The position of the popmenu can, but must not be on the anchor.
To position the menu on the ancher the left and top offset variables must be set to -1 or -2
-1 will align the left or top edge of the menu with the mouse pointer, -2 the center

Menu relative positioned to a div tag

If you want a popmenu positioned relative to a div tag you need to insert the following tag somewhere in your document:

<div id='MyTagID' style='position:relative'></div>
 
where MyTagID is referred to in the variable file by setting RelativeToLayer to "MyTagID";

The variables

Global variables

  • PopNoOffMenus - number - The number of Popmenus on the page.
  • BaseHref - Text string - Lets you specify the root directory for relative links. The script precedes your relative links with BaseHref.
    For instance:
    When you set BaseHref to "http://www.MyDomain/" and a link in the menu is "subdir/MyFile.htm", the script renders the link to: "http://www.MyDomain/subdir/MyFile.htm"
    Can also be used when you use images in the textfields of the menu "MenuX=new Array("<img src='"+BaseHref+"MyImage'>"....
    While testing the site on your harddisk use syntax like: var BaseHref="file:///C|/MyFiles/Homepage/"
  • PopRClick - number - Defines which menu reacts on right mouse click. Zero is none.
  • PopMenuSlide - Text string - Adds a graphic effect to the menu. Is only supported by MS IE6+.
    If you don't need MenuSlide just leave in the line var MenuSlide=""; and delete the other MenuSlide statements
    In general leave the MenuSlide you need in and delete the others.
  • PopMenuShadow - Text string - Adds a graphic effect to the menu. Is only supported by MS IE6+.
    If you don't need MenuShadow just leave in the line var MenuShadow=""; and delete the other MenuShadow statements
    In general leave the MenuShadow you need in and delete the others.
  • PopMenuOpacity - Text string - Adds a graphic effect to the menu. Is only supported by MS IE6+.
    If you don't need MenuOpacity just leave in the line var MenuOpacity=""; and delete the other MenuOpacity statement
    In general leave the MenuOpacity you need in and delete the other.

Functions

  • BeforeStart - Is called just before the menu is build. Is by default empty. Can be used by user.
  • AfterBuild - Is called just after the menu is build. Is by default empty. Can be used by user.
  • BeforeFirstOpen - Is called when the mouse enters a main menu item that has children. Is by default empty. Can be used by user.
  • AfterCloseAll - Is called when the mouse leaves the menu. Is by default empty. Can be used by user.

Variables per Menu

Each Popmenu is controlled by the array PopMenuX, where X is the number of the PopMenu
The first PopMenu is controlled by the array PopMenu1. The second PopMenu is controlled by the array PopMenu2....

The elements of the array PopMenuX are:

  • NoOfFirstLineElements - number - The number of primary elements of the Popmenu.
  • Left - number - Defines the left coordinate of the Popmenu. Can be positive or negative.
    There are two special values that are only usefull when the menu is triggered by moving the mouse over an anchor: -1 places the left side of the Popmenu at the same coordinate as the mouse pointer, -2 places the horizontal center of the Popmenu at the location of the mouse pointer.
  • Top - number - Defines the top coordinate of the Popmenu. Can be positive or negative.
    There are two special values that are only usefull when the menu is triggered by moving the mouse over an anchor: -1 places the top side of the Popmenu at the same coordinate as the mouse pointer, -2 places the vertical center of the Popmenu at the location of the mouse pointer.
  • FontColor - Text string - Font color when the mouse is not over the element.
    Can be a supported color name like red or blue or a RGB string like #ff552a.
  • BgColor - Text string - Background color when the mouse is not over the element.
    Can be a supported color name like red or blue or a RGB string like #ff552a.
  • FontHighColor - Text string - Font color when the mouse is over the element.
    Can be a supported color name like red or blue or a RGB string like #ff552a.
  • BgHighColor - Text string - Background color when the mouse is over the element.
    Can be a supported color name like red or blue or a RGB string like #ff552a.
  • BorderColor - Text string - Border color of the main menu.
    Can be a supported color name like red or blue or a RGB string like #ff552a.
  • FontFamily - Text string - More than one font can be declared seperated with a comma.
    Example: arial, times new roman, ms comic sans. The browser will use the first font found on the users system.
  • Bold - Number 1 or 0 - 1 makes the font weight bold; 0 makes the font weight normal.
  • Italic - Number 1 or 0 - 1 makes the font italic; 0 makes the font normal.
  • FontSize - Number - Size of the font in pt.
  • FirstLineHorizontal - Number 1 or 0 - When set to 1 the elements of the first level are placed horizontal next to each other.
    When set to 0 these elements are placed vertical below each other.
  • PermanentVisible - Number 1 or 0 - When set to 1 the elements of the first level are always visible.
  • BorderWidth - Number - Thickness of the border around the elements in pixels.
  • TextAlign - Text string. Can be "left", "center" or "right".
  • HorizontalOverlap - Number between 0 and 1 - Controls what part of a level is covered with it's sublevel.
    With a value of .25 a level will covered be for 25% by it's sublevel. Negative values are possible, but not very usefull.
  • VerticalOverlap - Number between 0 and 1 - Controls the vertical offset of a sublevel from it's parent level.
    With a value of .25 a sublevel will start 25% of the parents level height lower. Negative values will make the start higher.
  • UnfoldDelay - Number - The time in milliseconds the menu's sublevels stay visible after the mouse is no longer over the menu.
  • UnfoldRightToLeft - Number 1 or 0 - Controls if the menu unfolds right to left or left to right
  • RelativeToLayer - Text string- Enables relative positioning of the menu.
    1- In the HTML-file where the menu shows you need a named div:
    <div id='MenuPos' style='position:relative'></div>
    2- Set RelativeToLayer to 'MenuPos'
    3 - Set MenuCentered to 'left', MenuVerticalCentered to 'top' and Top and Left to 0
    4 - Top, Left and the Menu center variables can still be used to get an offset from the <div>.
     
    Remarks: the different ways browsers interpret the pageX and pageY offset makes this feature difficult. I got the best results by putting the div inside a table, give the div the same dimensions as the first level menu, put an transparent image inside the div with again the same dimensions as the first level menu
    <table>
      <tr><td>
        <div id='MenuPos' style='position:relative; width:102; height:102;'><img src='transparent.gif' width='102' height='102'></div>
      </td></tr>
    </table>
  • BorderBetweenElements - Number 1 or 0 - Controls if the border is only outside the elements or also in between the elements
  • MenuCentered - Text string - Controls the basic horizontal position of the menu. Can be 'left', 'center' or 'right'
  • MenuVerticalCentered - Text string - Controls the basic vertical position of the menu. Can be 'top', 'middle' or 'bottom'
  • ArrowRightSrc - Text string - Specifies image for arrow pointed right.
  • ArrowRightWidth - Number - The width of the Arrow Right image
  • ArrowRightHeight - Number - The height of the Arrow Right image
  • ArrowDownSrc - Text string - Specifies image for arrow pointed down.
  • ArrowDownWidth - Number - The width of the Arrow Down image
  • ArrowDownHeight - Number - The height of the Arrow Down image
  • ArrowLeftSrc - Text string - Specifies image for arrow pointed left.
  • ArrowLeftWidth - Number - The width of the Arrow Left image
  • ArrowLeftHeight - Number - The height of the Arrow Left image
  • TopPadding - Number - Space between top of element and it's content
  • LeftPadding - Number - Space between left of element and it's content
  • UnfoldsOnClick - Number - When set to 1 the menu unfolds onclick, when set to 0 the menu unfolds on mouse over.

Variables per Menu item

Elements in the same row may have different widths. Elements in the same collumn may have different heights.

  • ElementText - Text string - What you want to show in the element. It can be text, an image or infact any part of a html document including tags
    To show an image it must look like "<img src='MyImage' width='100' height='100'>"
    To use a rollover "rollover?Image1?Image2"
  • ElementLink - Text string - Where you want to go when you click the element.
    Looks like "MyLink.htm"
    When you want to link to an URL outside your own pages it must be "javascript:top.document.location.href='OutsideLink.htm';"
    or "javascript:window.open('OutsideLink.htm','_blank')"
    In fact with the help of javascript: you can start a whole script when the element is clicked. "javascript:{your script; another function;}"
  • ElementNoOffSubElements -Number - The number of sub elements of this element in the next level.
  • ElementHeight - Number - The pixelheight of the element.
  • ElementWidth - Number - The pixelwidth of the element.
  • ElementBgColor - Text string - When used it overrules the global variable Low Backcolor
    When not used it must be "".
  • ElementBgHighColor - Text string - When used it overrules the global variable High Backcolor
    When not used it must be "".
  • ElementFontColor - Text string - When used it overrules the global variable Font color
    When not used it must be "".
  • ElementFontHighColor - Text string - When used it overrules the global variable Font high color
    When not used it must be "".
  • ElementBorderColor - Text string - When used it overrules the global variable Border color
    When not used it must be "".
  • ElementFontFamily - Text string - When used it overrules the global variable Fontfamily
    When not used it must be "".
  • ElementFontSize - Number- When used it overrules the global variable FontSize
    When not used it must be -1.
  • ElementBold - Number- When used it overrules the global variable FontBold
    When not used it must be -1.
  • ElementItalic - Number- When used it overrules the global variable FontItalic
    When not used it must be -1.
  • ElementTextAlign - Text string - When used it overrules the global variable TextAlign
    When not used it must be "".
  • ElementStatusText - Text string - When used it overrules the normal use of the status bar
    When not used it must be "".

The menu tree

As explained above each independent Menu on the page is controlled by the array PopMenuX where X=1,2,3,....
Each PopMenuX has is own tree of levels.

  • PopMenu1_1, PopMenu1_2, PopMenu1_3, PopMenu1_n are first level elements of PopMenu1
  • PopMenu2_1, PopMenu2_2, PopMenu2_3, PopMenu2_n are first level elements of PopMenu2.
  • PopMenu1_1_1, PopMenu1_1_2, PopMenu1_1_3, PopMenu1_1_n are second level elements of PopMenu1_1

A possible menu tree for PopMenu1 might look like:
 

PopMenu1_1the first element of the first level of PopMenu1
PopMenu1_1_1the first sub element of PopMenu1_1
PopMenu1_1_2the second sub element of PopMenu1_1
PopMenu1_1_2_1the first sub element of PopMenu1_1_2
PopMenu1_1_2_2the second sub element of PopMenu1_1_2
PopMenu1_1_3the third sub element of PopMenu1_1
PopMenu1_1_3_1the first sub element of PopMenu1_1_3
PopMenu1_1_3_1_1the first sub element of PopMenu1_1_3_1
PopMenu1_1_3_1_2the second sub element of PopMenu1_1_3_1
PopMenu1_2the second element of the first level of PopMenu1
PopMenu1_3the third element of the first level of PopMenu1
PopMenu1_3_1the first sub element of PopMenu1_3
PopMenu1_3_1_1the first sub element of PopMenu1_3_1
PopMenu1_3_2the second sub element of PopMenu1_3
